## 1. 项目简介

### 1.1 这是什么？

一个**智能冰箱食材识别与管理系统**。简单来说就是：

- 冰箱里装一个摄像头，能自动识别冰箱里有什么食材
- 开门时摄像头开始工作，关门后自动分析拿了什么/放了什么
- 冰箱上的小屏幕可以查看食材清单和操作记录
- 电脑或手机上打开网页，也能随时查看冰箱里有什么

### 1.2 工作流程

```
你打开冰箱门
    → 摄像头启动，AI开始识别冰箱里的东西
    → 你拿出一个苹果（或放进去一瓶牛奶）
    → 你关上冰箱门
    → AI对比"开门前"和"开门后"的画面
    → 发现苹果不见了 → 记录"取出 苹果 x1"
    → 屏幕和网页上立即更新
```

### 1.3 核心功能

| 功能 | 说明 |
|------|------|
| 食材识别 | 使用YOLOv5 AI模型，能识别苹果、香蕉、牛奶、鸡蛋等常见食材 |
| 门磁感应 | 通过GPIO传感器感知冰箱门开关状态 |
| 自动灯光 | 开门自动亮灯（继电器控制） |
| LCD屏幕 | 480x480触摸屏，显示库存、操作记录，支持触屏编辑 |
| 网页管理 | 任何设备打开浏览器就能查看和管理食材 |
| 视频直播 | 开门时可以在网页上看冰箱实时画面 |

### 2.3 门磁传感器原理

门磁传感器就是一个磁控开关：

```
磁铁靠近 → GPIO读到高电平(1) → 门关着
磁铁远离 → GPIO读到低电平(0) → 门开着
```

在我们的开发板上，**GPIO32 = 0 表示门开着，GPIO32 = 1 表示门关着**。

## 6. 核心原理讲解

### 6.1 YOLOv5 食材识别

YOLOv5是一种目标检测算法，可以在一张图片中同时找出多个物体，并告诉你每个物体的位置和类别。

```
输入：一张图片 (640x640)
  ↓ NPU做AI推理
输出：检测到的物体列表
  [
    {"name": "苹果", "confidence": 0.95, "box": [100, 200, 150, 260]},
    {"name": "牛奶", "confidence": 0.88, "box": [300, 150, 380, 250]},
  ]
```

**关键技术点**：

1. **Letterbox缩放**：摄像头画面是720×480，但模型要求640×640输入。我们用letterbox方式缩放——等比缩放后放入640×640的画布（黑边填充），保证物体不变形。

2. **坐标映射**：模型输出的是640×640空间里的坐标，需要映射回实际的720×480画面。我们的 `mapCoordinates()` 函数就是干这个的。

3. **NPU加速**：AI推理不在CPU上跑（太慢），而是在专用的NPU（神经网络处理器）上跑。RV1106G3的NPU有1TOPS算力，跑YOLOv5n约5-10帧/秒。

### 6.2 食材变化检测

核心思路是**对比开门前后的检测结果**：

```
开门时（baseline）:
  {"苹果": 2, "牛奶": 1, "鸡蛋": 3}

关门时（after）:
  {"苹果": 1, "牛奶": 1, "面包": 1}

对比结果:
  苹果 2→1 → 取出 苹果 x1
  鸡蛋 3→0 → 取出 鸡蛋 x3
  面包 0→1 → 放入 面包 x1
```

代码实现在 `fridge_mgr.py` 的 `process_events()` 函数中。

### 6.3 视频流：从摄像头到浏览器

视频数据要经过多步转换才能在你的浏览器上播放：

```
摄像头输出：NV12格式（YUV的一种，节省带宽）
     ↓ OpenCV cvtColor (COLOR_YUV2RGB_NV12)
RGB格式（标准三通道彩色）
     ↓ YOLOv5画框（直接在RGB上画）
带标注的RGB图像
     ↓ VENC硬件编码（H264）
H264视频帧
     ↓ RTSP推流（网络协议）
RTSP视频流 (rtsp://192.168.2.77:554/live/0)
     ↓ ffmpeg转码（电脑上）
MJPEG图片流
     ↓ 浏览器 <img> 标签刷新
网页实时显示
```

**一个重要坑**：NV12→RGB的颜色转换用的是 `COLOR_YUV2RGB_NV12`（注意是RGB，不是BGR）。如果颜色不对（蓝色变黄色），问题通常出在这个转换上。

### 6.4 GPIO控制

```
GPIO32 → 输入模式 → 读取门磁状态 (0=开门, 1=关门)
GPIO40 → 输出模式 → 控制继电器   (1=灯亮, 0=灯灭)
```

在Python中用 `python-periphery` 库：

```python
from periphery import GPIO
door = GPIO(32, "in")    # GPIO32设为输入
relay = GPIO(40, "out")  # GPIO40设为输出

door_open = not door.read()  # 注意取反：0=开门
relay.write(True)             # 开灯
```

### 6.5 LCD屏幕绘制

开发板有一个480×480的屏幕，对应Linux设备文件 `/dev/fb0`（framebuffer）。

**画图的原理**：往 `/dev/fb0` 写入像素数据，屏幕就会显示对应的画面。

**像素格式**：每个像素占4个字节（32位），顺序是 B、G、R、A（蓝、绿、红、透明度）。

我们用Python的Pillow库画图：
1. 创建480×480的RGBA图像
2. 用ImageDraw在上面画文字、按钮、线条
3. 转换成BGRA字节序
4. 写入 `/dev/fb0`

**AI程序显示 vs Python程序显示的协调**：

- 开门时：AI程序往 `/dev/fb0` 写摄像头画面（=AI开，Python不抢）
- 关门时：Python管理程序往 `/dev/fb0` 写库存界面（=AI关，Python控制）

用 `/tmp/fb_lock` 文件做互斥锁：
- Python要写屏前创建 `/tmp/fb_lock` → AI看到锁就不写了
- 开门时Python删掉 `/tmp/fb_lock` → AI可以写摄像头画面

### 6.6 触摸屏

触摸屏型号是 Goodix GT911，使用Linux MT协议B（多点触控协议）。

触摸事件在 `/dev/input/event0` 设备上读取：

```
事件格式: 16字节 = 时间戳(8) + 类型(2) + 编码(2) + 值(4)
类型=3 编码=53 → X坐标 (0-480)
类型=3 编码=54 → Y坐标 (0-480)
类型=3 编码=57 → 触摸ID (-1=抬起, >=0=按下)
类型=0 编码=0  → SYN_REPORT (一帧结束)
```

**触摸区域检测**：在画图时记录每个按钮的坐标范围 `(x, y, width, height)`，触摸事件来临时判断触点落在哪个按钮范围内。

## 7. 交叉编译指南

### 7.1 什么是交叉编译？

开发板的CPU是ARM架构，你的电脑是x86架构。在x86电脑上编译出能在ARM开发板上运行的程序，叫做**交叉编译**。

### 7.2 编译步骤

所有步骤在**WSL Ubuntu**中执行：

```bash
# 1. 进入SDK目录
export LUCKFOX_SDK_PATH=/home/taki/luckfox-pico

# 2. 进入示例源码目录
cd /home/taki/luckfox_pico_rkmpi_example

# 3. 创建并进入编译目录
mkdir -p build_merged && cd build_merged

# 4. CMake配置
cmake .. \
  -DLIBC_TYPE=uclibc \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_C_COMPILER=arm-rockchip830-linux-uclibcgnueabihf-gcc \
  -DCMAKE_CXX_COMPILER=arm-rockchip830-linux-uclibcgnueabihf-g++ \
  -DEXAMPLE_DIR=example/luckfox_pico_rtsp_yolov5 \
  -DEXAMPLE_NAME=fridge_ai

# 5. 编译（8线程加速）
make -j8

# 6. 编译成功后会生成 fridge_ai 文件
ls -lh fridge_ai
# 输出类似: -rwxr-xr-x fridge_ai (约4MB)
```

### 7.3 复制到Windows

编译好的文件在WSL中，需要复制到Windows目录：

```powershell
# 在PowerShell中
copy "\\wsl$\Ubuntu-22.04\home\taki\luckfox_pico_rkmpi_example\build_merged\fridge_ai" "c:\claude_code_demo\deploy\"
```

### 7.4 部署到开发板

```bash
# 将文件通过SSH传到开发板
# 在PowerShell或WSL中
scp c:\claude_code_demo\deploy\fridge_ai root@192.168.2.77:/root/smartfridge/bin/

# 确认传输成功
ssh root@192.168.2.77 "ls -la /root/smartfridge/bin/"
```

### 9.2 访问方式

| 方式 | 地址 | 说明 |
|------|------|------|
| 本机 | http://127.0.0.1:5000 | 运行Flask的电脑自己访问 |
| 局域网 | http://192.168.2.75:5000 | 同WiFi下的其他设备 |
| 公网 | https://xxxxxx.loca.lt | 任何有网络的地方都能访问 |

### 9.3 公网访问原理

```
你的电脑(Flask:5000端口)
    ↓ localtunnel连接
localtunnel.me 服务器
    ↓ 分配随机公网地址
https://xxxxxx.loca.lt
    ↓ 任何设备访问这个地址
实际连接到你的Flask服务器
```

注意：
- localtunnel分配的地址每次重启Flask都会变
- 首次访问需要输入你的公网IP地址来做验证
- 如果需要固定公网地址，可以用ngrok（需注册）

## 11. 常见问题排查

### Q: 视频流显示"Connecting..."或卡住

**可能原因和解决**：

```
1. 门关着 → AI没启动 → 没有视频流 → 正常现象，开门就好
   确认：ssh到开发板执行 cat /sys/class/gpio/gpio32/value
   输出 0 → 门开着（应该有问题）
   输出 1 → 门关着（正常，没视频流）

2. 门开着但没有视频 → AI程序没启动
   检查：ssh到开发板执行 ps aux | grep fridge_ai
   没看到进程 → 手动启动 /root/smartfridge/bin/fridge_ai /root/smartfridge/model/yolov5.rknn

3. 多个ffmpeg进程冲突
   解决：Windows上执行 taskkill /f /im ffmpeg.exe，然后重启Flask

4. RTSP端口被占用
   检查：ssh到开发板执行 netstat -tlnp | grep 554
   应该看到RTSP LISTENING，没有则重启管理程序
```

### Q: 颜色不对（蓝色变黄色）

这个bug已经被修复。如果重新出现，检查 `main.cc` 中的颜色转换：

```cpp
// ✓ 正确的写法
cv::cvtColor(yuv420sp, rgb, cv::COLOR_YUV2RGB_NV12);

// ✗ 错误（会导致R/B通道互换）
cv::cvtColor(yuv420sp, rgb, cv::COLOR_YUV2BGR_NV12);
```

### Q: 数据库不同步

```bash
# 检查开发板上的数据
ssh root@192.168.2.77 "cat /root/smartfridge/fridge_data.json"

# 检查服务器数据

# 手动触发同步
# 在开发板上重启管理程序即可

```

### Q: LCD触摸不响应

```bash
# 1. 检查触摸设备是否存在
ssh root@192.168.2.77 "ls -la /dev/input/event*"

# 2. 检查触摸驱动是否加载
ssh root@192.168.2.77 "cat /sys/class/input/event0/device/name"
# 应该显示 goodix 或 GT9xx

# 3. 如果用cat读取event0有乱码输出，说明硬件正常
ssh root@192.168.2.77 "cat /dev/input/event0 | hexdump -C"
# 然后触摸屏幕，应该有数据变化

# 4. 确认fridge_mgr在运行
ssh root@192.168.2.77 "ps aux | grep fridge_mgr"
```

### Q: 开发板内存不足

```bash

# 停掉不必要的服务
ssh root@192.168.2.77 "/oem/usr/bin/RkLunch-stop.sh"
```

### Q: SCP传文件失败 "No such file"

开发板上的目标目录不存在。先SSH登录创建：

```bash
ssh root@192.168.2.77
mkdir -p /root/smartfridge/bin /root/smartfridge/model /root/smartfridge/logs
```

### Q: Flask启动报错 "Address already in use"

5000端口被占用（通常是你上次的Flask没关）：

```powershell
# 找到并杀掉占用5000端口的进程
netstat -ano | findstr :5000
# 记下PID，然后
taskkill /f /pid 那个PID

# 或者直接杀掉所有python和ffmpeg
taskkill /f /im python.exe
taskkill /f /im ffmpeg.exe

# 然后重新启动Flask
cd c:\claude_code_demo\server
python app.py
```

### Q: localtunnel公网地址连不上

```bash
# 1. 确认localtunnel已安装
npm list -g localtunnel

# 2. 如果没有，安装
npm i -g localtunnel

# 3. 首次访问localtunnel地址时，需要验证
# 浏览器打开 https://xxxxxx.loca.lt
# 在 "Endpoint IP" 输入框里填你的公网IP
# 公网IP查询: https://whatismyipaddress.com
```
